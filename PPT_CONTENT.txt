## Slide 1: Title Slide
- **Project Name**: NeuralCredit_
- **Subtitle**: Advanced AI-Driven Loan Risk Assessment & Governance Platform
- **Core Value**: Bridging the gap between Machine Learning precision and Financial Governance.
- **Visuals**: Dark theme, Neural-Industrialist aesthetic, Infinity/Satellite logo motif.

---

## Slide 2: Problem Statement
- **The Gap**: Traditional loan processing is either too slow (manual) or too opaque (black-box automated).
- **Challenges**:
    - Limited data points (relying solely on CIBIL).
    - High latency in ML model loading (cold-start issues).
    - Lack of transparency for rejected applicants.
    - No built-in governance for anomalous/fraudulent applications.

---

## Slide 3: The Solution (NeuralCredit_)
- **Conceptual Overview**: A high-fidelity FinTech command center that leverages a persistent AI engine for instantaneous risk profiling.
- **Key Pillars**:
    1. Instant Inference: Sub-millisecond decision making.
    2. Explainable AI (XAI): Visualizing feature importance.
    3. Human-in-the-loop: Admin governance for anomaly review.
    4. Premium UX: Immersive "Neural" design language.

---

## Slide 4: Core Technology Stack
- **Frontend**: React 19 + Vite + Tailwind CSS + Framer Motion.
- **Backend**: Node.js/Express + Better-SQLite3 (WAL Mode).
- **AI Engine (Aura)**: Persistent Python process (keeps model in RAM).
- **Machine Learning**: Scikit-Learn (Random Forest) for multi-dimensional classification.

---

## Slide 5: Technical Innovation – The Aura Engine
- **The Problem**: Standard scripts load 100MB+ models per request (~10s latency).
- **The Innovation**:
    - Aura Engine stays resident in RAM as a background service.
    - Reduced latency from ~10,000ms to <50ms.
    - Communicates via low-level stdin/stdout streams for maximum speed.

---

## Slide 6: Key Features – User Experience
- **System Boot Sequence**: Immersive terminal-style entry animation.

- **Identity Verification**: Simulated biometric and document scanning nodes.

## Slide 7: Key Features – Governance & Admin
- **Threshold Tuning**: Real-time adjustment of approval sensitivity (e.g., 0.5 -> 0.7).
- **Anomaly Detection**: Automated flagging of unusual financial patterns.
- **Review Queue**: Dedicated interface for manual administrative oversight.
- **Global Stats**: Real-time distribution charts for CIBIL scores and employment.

## Slide 8: Security & Reliability
- **Authentication**: Secure session management with Bcrypt hashing.
- **RBAC**: Role-Based Access Control protecting administrative endpoints.
- **Rate Limiting**: Intelligent IP throttling to prevent model scraping.
- **Data Integrity**: SQLite Write-Ahead Logging (WAL) for safe concurrency.
